# 阶段 2 / 本地二维码图片导入状态

- 更新时间：2026-07-16
- 当前结论：已完成本地单张二维码图片选择、离线识别、TOTP 参数校验、重复检测、导入确认与加密保存闭环。剪贴板图片与链接导入已在 [阶段 3](./PHASE3_STATUS.md) 完成，macOS 区域截图已在 [阶段 4](./PHASE4_STATUS.md) 完成；Windows 自有区域截图已在 [阶段 5](./PHASE5_STATUS.md) 实现但待平台验证；Google Authenticator 多账号和多张迁移二维码已在 [阶段 6](./PHASE6_STATUS.md) 完成。

## 本阶段已完成

### 导入模型与安全边界

- [x] 建立统一的 `OtpImportCandidate`，保存已验证的账号草稿、导入来源与重复状态。
- [x] 图片文件仅在本地读取和解析，不上传、不调用网络服务。
- [x] 导入来源只展示安全文件名，不在界面或错误信息中暴露本机完整路径。
- [x] 将标准 `otpauth://totp` 内容转换为现有 `AccountDraft`，继续复用账号参数校验和 Vault 加密保存流程。
- [x] 非 TOTP、无二维码、无效 Secret 或不支持的参数均返回用户可理解的中文错误，不显示敏感原始数据。

### 图片选择与二维码识别

- [x] 使用系统文件选择器导入 PNG、JPEG、WebP、BMP、GIF 和 TIFF 图片。
- [x] 默认限制单个图片文件不超过 20 MB、解码后像素不超过 40 MP，避免异常图片造成过高内存占用。
- [x] 对大图按最长边 2400 像素缩放后识别，原图数据不会写入临时目录。
- [x] QR 解码运行在独立 Isolate，避免大图处理阻塞主界面。
- [x] 自动尝试原始方向、90°、180°、270°旋转以及反色图像，并启用 ZXing `tryHarder`。
- [x] 当前从一张图片中读取首个可识别二维码；同一图片中的多个独立二维码尚不能一次批量识别，但 Google Authenticator 多张迁移二维码可逐张读取并收集。

### 导入确认与账号保存

- [x] 添加账号菜单支持“手动输入或链接”和“从二维码图片导入”。
- [x] 图片解析期间显示“正在解析…”，并阻止重复触发导入。
- [x] 识别成功后进入统一确认页面，可检查和编辑发行方、账号名、Secret、算法、位数和周期。
- [x] Secret 默认隐藏，可由用户临时切换显示。
- [x] 对 issuer、账号名和 Secret 完全相同的账号给出重复提示；仅在用户明确勾选后允许保留副本。
- [x] 确认后的账号通过现有 Vault controller 加密保存，并立即刷新 TOTP 列表。
