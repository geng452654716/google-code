# 阶段 7 / 单账号安全分享状态

- 更新时间：2026-07-16
- 当前结论：已完成单账号 Secret、标准 `otpauth://` 链接和二维码的安全分享闭环；每次进入分享内容前必须重新验证主密码，材料仅在内存生成，并在复制、保存、超时、失焦、后台或 Vault 锁定时隐藏或关闭。

## 本阶段已完成

### 重新认证与入口

- [x] 每个账号的更多操作菜单增加“分享账号”。
- [x] 分享页先展示长期凭据泄露风险，不在重新认证前生成或展示 Secret、链接和二维码。
- [x] 每次分享都重新读取加密 Vault envelope，并用用户输入的主密码执行完整解密校验。
- [x] 重新认证不会替换当前已打开 Vault、不会修改会话 payload，也不会把密码或验证结果写入状态。
- [x] 错误密码只返回安全错误，不锁定当前会话，也不暴露底层加密诊断。

### 分享格式

- [x] 支持复制原始 Base32 Secret。
- [x] 支持复制标准 `otpauth://totp` URI。
- [x] URI 完整包含 issuer、账号名、算法、位数和周期。
- [x] 支持生成与 URI 内容一致的高对比度 PNG 二维码。
- [x] Secret 和 URI 默认遮挡，必须由用户主动点击显示。
- [x] 二维码只在成功重新认证后于内存中按需生成，不写入 Vault 或应用缓存。

### 敏感内容生命周期

- [x] 分享材料默认只保留 60 秒，并显示剩余时间。
- [x] 超时后释放当前分享材料，返回主密码重新认证页。
- [x] 窗口进入 inactive、hidden、paused 或 detached 时立即隐藏材料。
- [x] Vault 锁定时立即清除材料，并无动画移除高风险分享路由。
- [x] 复制 Secret 或 URI 后立即隐藏分享材料，并使用 30 秒敏感剪贴板 TTL。
- [x] 清理剪贴板前重新读取当前内容；用户之后复制了其他内容时不会误清除。
- [x] 敏感剪贴板服务提升为应用级 Riverpod Provider，账号页因锁定而销毁时不会取消尚未执行的清理任务。

### 二维码保存

- [x] 使用系统保存对话框让用户明确选择 PNG 路径。
- [x] 直接写入用户选择的位置，不创建应用临时文件或隐藏副本。
- [x] 建议文件名会清理 Windows/macOS 非法字符并限制长度。
- [x] 保存成功后立即隐藏分享材料。
- [x] 打开系统保存对话框造成应用失焦时，界面按安全优先策略立即遮挡；已由用户发起的本次保存仍可使用局部内存字节完成。

## 当前限制与风险

- [ ] 首版尚未接入 macOS NSSharingServicePicker 或 Windows DataTransferManager；复制和用户选择路径保存已可用。
- [ ] 敏感操作重新认证当前只支持主密码，Touch ID / Windows Hello 留待后续阶段。
- [ ] 二维码保存路径和文件权限仍需在 macOS sandbox、Windows 10 与 Windows 11 真机验证。
- [ ] Dart 垃圾回收无法保证 Secret 所在内存立即物理清零；当前通过缩短引用和展示生命周期降低风险。
- [ ] Secret 一旦发送给其他人或设备，本应用无法撤销；必须到原服务重新绑定二次验证才能使旧 Secret 失效。
