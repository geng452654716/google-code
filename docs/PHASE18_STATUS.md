# 阶段 18：账号分组与拖拽归类

- 完成日期：2026-07-20
- 范围：本地分组创建、重命名、删除、筛选，以及账号拖拽归类
- 数据兼容：保持 `VaultPayload.schemaVersion == 1`

## 已完成

1. 在验证码首页增加固定分组侧栏，提供“全部账号”“未分组”和自定义分组。
2. 支持创建、重命名和删除分组：
   - 名称自动去除首尾空白；
   - 名称最长 40 个 Unicode 字符；
   - 名称按不区分大小写的方式拒绝重复。
3. 每张账号卡片增加独立拖拽手柄，可拖入自定义分组或“未分组”。
4. 分组筛选可与现有账号搜索组合使用，并显示各分组账号数量。
5. 删除分组不会删除验证码账号；关联账号会在同一次 Vault 保存中移到“未分组”。
6. 分组和账号关联仍写入本地加密 Vault，不新增网络、后端或明文存储。

## 一致性与失败处理

- 新建、重命名、删除和移动操作都由 `VaultSessionController` 统一提交。
- 删除分组与清空账号 `groupId` 是一个原子 payload 变更。
- Repository 保存成功后才替换会话 payload；失败时显示错误并保留原数据。
- 无效目标分组、未知账号和重复名称不会触发保存。
- “未分组”仅是 UI 虚拟项，持久化表达为 `Account.groupId == null`。
